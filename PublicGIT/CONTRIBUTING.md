# Contributing to GitCommitter

First off, thank you for considering contributing to GitCommitter! It's people like you that make GitCommitter such a great tool for the developer community.

## Code of Conduct

By participating in this project, you agree to abide by our [Code of Conduct](./CODE_OF_CONDUCT.md).

## How Can I Contribute?

### Reporting Bugs

-   **Check the Issues Tab**: Someone may already have reported the bug.
-   **Open a New Issue**: If you can't find an existing issue, open a new one. Include as much detail as possible, such as steps to reproduce, expected behavior, and actual behavior.

### Suggesting Enhancements

-   **Open an Issue**: Clearly describe the enhancement you'd like to see and why it would be beneficial.

### Pull Requests

1.  **Fork the Repository**: Create your own fork of the project.
2.  **Create a Branch**: Use a descriptive name for your branch (e.g., `fix/typo-in-readme`).
3.  **Make Your Changes**: Ensure your code follows our style guidelines.
4.  **Test Your Changes**: Run any existing tests and add new ones if necessary.
5.  **Submit a Pull Request**: Provide a clear description of your changes and link to any related issues.

## Style Guidelines

-   Use TypeScript for all new code.
-   Follow the existing code structure and naming conventions.
-   Write clear, concise commit messages.

## Questions?

If you have any questions, feel free to open an issue or contact the maintainers.
